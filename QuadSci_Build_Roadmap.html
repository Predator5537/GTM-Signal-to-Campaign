What You're Building (Plain English)

You are building a system that runs on its own, 24 hours a day, doing 4 things automatically:

  1. Watching the internet for signs that a company might be ready to buy QuadSci (job postings, G2 reviews, executive LinkedIn posts, funding news).
  2. Scoring each company using a 100-point rubric — bigger companies with more signals get higher scores. Only companies scoring 60+ move forward.
  3. Writing a personalized email for each qualified company using AI — an email that references the exact signals you found, not a generic template.
  4. Either sending the email automatically (for mid-tier accounts) or putting it in a queue for you to approve with one click (for top accounts).

Your only daily job: spend 20–30 minutes reviewing and approving the top accounts in a Google Sheet. Everything else runs itself.

✅ Total cost: $0/month. Every tool used has a free tier that covers this workflow's volume (40–80 accounts/month).

Architect's PRD Audit — 5 Issues Fixed Before You Build

Before you build anything, you need to know these issues that exist in the PRD. I've already fixed them in the build plan below — you don't have to do anything differently, but understanding why a step looks a certain way will save you confusion.

ISSUE #1 · CRITICAL

Railway's free tier no longer exists

The PRD says "self-host n8n on Railway free tier." Railway removed their free tier in August 2024.

✓ FIX: We'll use n8n.cloud (14-day free trial, no credit card needed). It's simpler, no server to manage, and the trial covers the whole exercise.

ISSUE #2 · CRITICAL

Apify's free tier only allows 5 Actor runs/month — not daily

The PRD describes daily LinkedIn + G2 scraping. Apify's free tier gives you 5 runs/month total. Daily scraping would exhaust that in 2.5 days.

✓ FIX: We use Apify once per week (Monday morning). Daily monitoring uses Google Alerts RSS feeds, which are unlimited and free. Weekly Apify runs catch what RSS misses.

ISSUE #3 · MODERATE

Apollo.io free tier doesn't give full API access for sequence enrollment

The PRD describes n8n automatically enrolling contacts into Apollo sequences via API. Apollo's API for sequence enrollment requires at least the Basic plan ($49/mo).

✓ FIX: n8n generates the email and stages it in Google Sheets. For Tier 2, you create a sequence in Apollo's web UI manually (10 minutes of setup, then hands-off). For Tier 1, n8n sends via Gmail directly — which actually has better deliverability for high-value targets anyway.

ISSUE #4 · MINOR

The PRD mentions a "Python script" for copy generation — not needed

Section 8.4 describes a Python script to call the AI. You don't need to write any Python. n8n has a built-in AI node that does this with zero code.

✓ FIX: Everything runs inside n8n workflows. No Python, no terminal, no code files.

ISSUE #5 · MINOR

The Google Sheets schema is not defined in the PRD

The PRD says "use Google Sheets" but doesn't specify the exact columns. n8n needs to write to specific columns. If the columns don't match, nothing gets saved correctly.

✓ FIX: Phase 1 of this guide gives you the exact, ready-to-create Google Sheet with every column named and explained. Copy it exactly.

// Groups all signals by company domain
// Returns only companies with 3+ signals within 21 days
const items = $input.all();
const now = new Date();
const cutoff = new Date(now - 21 * 24 * 60 * 60 * 1000); // 21 days ago

const groups = {};
for (const item of items) {
  const domain = item.json['Company Domain'];
  const date = new Date(item.json['Date']);
  if (!domain || date < cutoff) continue;

  if (!groups[domain]) {
    groups[domain] = {
      domain,
      name: item.json['Company Name'],
      signals: [],
      types: new Set()
    };
  }
  groups[domain].signals.push(item.json['Signal Summary']);
  groups[domain].types.add(item.json['Signal Type']);
}

const clusters = Object.values(groups)
  .filter(g => g.signals.length >= 3)
  .map(g => ({
    company_domain: g.domain,
    company_name: g.name,
    signal_count: g.signals.length,
    signal_types: [...g.types].join(','),
    signal_summaries: g.signals.join(' | ')
  }));

return clusters.map(c => ({ json: c }));
