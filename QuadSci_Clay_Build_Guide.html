Clay Table — Column-by-Column Build Guide

QuadSci GTM Demo  ·  For someone with some Clay experience  ·  Build time: 2–2.5 hours  ·  Free tier safe

Before you start: Complete the Google Sheets workbook first (run createQuadSciWorkbook() in QuadSci_SheetSetup.gs). You need the spreadsheet URL before Step 1.
Clay free tier credit budget: 100 Data Credits/month. The demo uses approximately 24 credits total (3 accounts × ~8 credits each for Find Company + Find Person + Find Email). You have enough headroom for 1–2 demo runs. Do NOT click "Enrich All" on a column until you have checked the conditional logic is set.

Step 0 — Create the Clay Table

SETUP — One-time actions before adding any columns
1
Go to clay.com → New Table → Import from Google Sheets
2
Connect your Google account and select the spreadsheet created by QuadSci_SheetSetup.gs. Choose the Accounts tab (not Signal Log or Send Queue).
3

4
Name the table: "QuadSci Pipeline"
5
After import, you should see 3 rows: acme.com, betatech.com, gammasaas.com. Confirm all 13 columns from the Accounts sheet are present as Clay columns.
If Clay imports extra blank rows: Filter by domain ≠ empty. Clay sometimes creates placeholder rows from empty sheet rows.

Phase 1 — Cluster Detection (Formula Columns)

These columns compute from data already in the sheet. Zero enrichment credits. Build these first.

14
cluster_qualified Formula

What it does: Returns TRUE only if an account has ≥3 signals AND at least one anchor signal. This is the gate before any enrichment runs.

How to add in Clay: Add Column → Formula → paste this:

{signal_count} >= 3 && {has_anchor} == true
Note: In Clay's formula editor, column names are wrapped in curly braces. {signal_count} references the signal_count column imported from Google Sheets. Make sure the column names match exactly (case-sensitive in some Clay versions).
15
icp_pass Formula

What it does: Hard-filter before spending any enrichment credits. An account that fails ICP will never consume a Find Person or Find Email credit.

{employees} >= 200 && {industry} == "B2B SaaS" && {cluster_qualified} == true

Note: funding_stage check is omitted from the formula because the Accounts sheet already filtered to Series B+. Add it back if you expand the signal sources beyond the demo data.

Phase 3 — Scoring (Formula Columns)

All scoring is pure arithmetic. Zero AI credits. Build after enrichment columns exist.

19
firmographic_score Formula

Max: 40 points. Components: employees (20 pts max), funding stage (15 pts max), ARR (15 pts max).

(
  {employees} >= 500 ? 20 :
  {employees} >= 200 ? 15 :
  {employees} >= 100 ? 8 : 0
)
+
(
  {funding_stage} == "Series C" || {funding_stage} == "Series D" || {funding_stage} == "Series E" ? 15 :
  {funding_stage} == "Series B" ? 10 : 0
)
+
(
  {arr_estimate} >= 100000000 ? 15 :
  {arr_estimate} >= 50000000 ? 8 : 0
)

Clay formula ternary syntax: condition ? value_if_true : value_if_false. Chain with parentheses + plus sign.

20
signal_score Formula

Max: 45 points. Checks which signal types fired using the signal_types column (comma-separated string like "S1,S2,S5").

(CONTAINS({signal_types}, "S5") ? 20 : 0)
+
(CONTAINS({signal_types}, "S2") ? 15 : 0)
+
(CONTAINS({signal_types}, "S3") ? 15 : 0)
+
(CONTAINS({signal_types}, "S4") ? 10 : 0)
+
(CONTAINS({signal_types}, "S1") ? 10 : 0)
+
({signal_count} > 3 ? ({signal_count} - 3) * 5 : 0)
Clay CONTAINS syntax: CONTAINS(text, substring) returns true/false. This checks if "S5" appears in the signal_types string. Works correctly with comma-separated values.
21
tech_score Formula

Max: 15 points. Based on tech_stack column (comma-separated string).

(CONTAINS({tech_stack}, "Gainsight") || CONTAINS({tech_stack}, "ChurnZero") ? 10 : 0)
+
(CONTAINS({tech_stack}, "Mixpanel") || CONTAINS({tech_stack}, "Segment") ||
 CONTAINS({tech_stack}, "Amplitude") ? 5 : 0)
+
(CONTAINS({tech_stack}, "Salesforce") ? 5 : 0)

Note: capped at 15 even if both lines fire. If you want a hard cap, wrap in MIN(15, ...).

22
total_score Formula 
{firmographic_score} + {signal_score} + {tech_score}

Expected demo values: Acme Corp ~83, Beta Tech ~68, Gamma SaaS ~47

23
tier Formula 
{total_score} >= 80 ? "Tier 1" :
{total_score} >= 60 ? "Tier 2" :
{total_score} >= 40 ? "Tier 3" : "Discard"

Expected: Acme Corp = Tier 1, Beta Tech = Tier 2, Gamma SaaS = Tier 3

Phase 4 — AI Copy Generation

Uses Clay's AI Column. Model: Claude 3.5 Sonnet (select from Clay's model picker). Two AI columns: email_subject and email_body. Both conditional on tier ≠ "Discard" AND buyer_found = true.

AI credits: Clay's free tier includes AI Column credits. Claude 3.5 Sonnet costs roughly 1 Clay credit per AI Column call. With 3 accounts × 2 AI Columns each = ~6 AI calls. Well within free tier. Only run AI columns AFTER scoring is complete and tiers are confirmed.

Account: {{company_name}} ({{domain}})
Buyer title: {{find_person.title}}
Tier: {{tier}}
Signals fired: {{signal_types}}

Write ONE subject line only. 4 to 7 words. No punctuation at end.

Rules:
- Tier 1: Reference the strongest anchor signal (earnings NRR drop or G2 churn review) specifically. Be direct.
- Tier 2: Reference the buyer's tech stack pattern (e.g. "The Gainsight to Snowflake handoff problem"). Do NOT name the specific signal.
- Tier 3: Write a pattern observation subject, not a signal callout.
- No exclamation points. No emojis. No "Quick question". No "Following up".
- Output the subject line text only. Nothing else.

[CONTEXT]
You are a B2B outbound sales copywriter writing a first-touch plain-text email

[QUADSCI FACTS — use only these, invent nothing else]
- Predicts customer churn 12 to 18 months before renewal with 94% accuracy.
- Ingests raw product telemetry (trillions of events), not CRM-derived data.
- Statistically grounded, explainable machine learning — not black-box scoring.
- Two products: Growth AI (churn and expansion prediction) and Cohorts AI
  (statistically significant usage-pattern segments).
- Real customers: Clari, Reltio. Integrates with 50+ tools including
  Salesforce and Mixpanel.

[INPUTS]
company_name: {{company_name}}
buyer_name: {{find_person.name}}
buyer_title: {{find_person.title}}
tier: {{tier}}
signal_summaries: {{signal_summaries}}
tech_stack: {{tech_stack}}

[TIER-SPECIFIC OBJECTIVE]
If tier is Tier 1:
  - Subject line is 4 to 7 words.
  - Body is 80 to 120 words (excluding greeting).
  - Open with the strongest fired signal from signal_summaries, named by source
    (the G2 review, the earnings call, the job posting, the LinkedIn post).
  - Connect that signal to a QuadSci outcome relevant to the buyer title.
  - End with: "Surprise churn is a visibility problem, not a CSM skill problem.
    Open to 20 minutes this week?"
  - Tone: peer-to-peer, direct. Assume the pain is acute.

If tier is Tier 2:
  - Body is 70 to 100 words (excluding greeting).
  - Open with a pattern observation (NOT a direct callout of the specific signal).
    Example: "Most CS-Ops teams running Gainsight on Snowflake hit the same wall..."
  - Bridge to QuadSci as one data point, not the whole answer.
  - End with one of: "Happy to share how Clari is handling this if useful."
    OR "Curious if this maps to what you're seeing internally."
  - Tone: observational, educational. Do not assume the pain is acute.

If tier is Tier 3:
  - Body is 60 to 80 words (excluding greeting).
  - Open with an industry observation about NRR visibility at their stage.
  - Mention QuadSci briefly as context.
  - End with a content offer or a low-stakes question. No meeting request.
  - Tone: informational. No urgency.

[PERSONA GUIDANCE — match to buyer_title]
If title contains CRO or Chief Revenue or VP Revenue or Head of Revenue:
  Emphasize forecast accuracy you can defend to the board; NRR as a
  predictable metric; preempting expansion gaps.

If title contains CS or Customer Success or Chief Customer:
  Emphasize giving CSMs runway to act, not react; removing surprise churn
  from QBRs; shifting CSM work from triage to expansion.

If title contains RevOps or Revenue Operations or CS Operations:
  Emphasize telemetry as a first-class data source (not stitched from CRM);
  reducing scoring fragility tied to CRM hygiene.

Default if none match: CS leadership framing.

[HARD RULES]
- Greeting must be exactly "Hi [first name of buyer_name],"
- Mention company_name naturally in sentence 1 or 2.
- Signature is added separately — do NOT include a signature.

[STRICT PROHIBITIONS]
- NO em-dash characters. Replace with commas, periods, or regular hyphens.
- NO exclamation points. NO emojis. NO slang.
- NO phrases: "I hope this finds you well", "just checking in",
  "circling back", "touching base", "wanted to reach out", "quick question".
- NO words: "game-changer", "revolutionize", "unlock", "supercharge",
  "leverage", "synergy", "seamless".
- NO invented numbers. Do not state employee count, ARR, CS team size, or
  any figure unless it is in the inputs. Do not infer or estimate.
- NO invented customer quotes. Only reference Clari and Reltio as customers.
- NO bullet points. NO markdown. NO links. NO placeholders.
- If an input is empty, write around it. No empty brackets or N/A.

[OUTPUT FORMAT — return the email body only, no subject line, no signature]
Hi [first_name],

[paragraph 1]

[paragraph 2]

[closing line from tier-specific objective above]

Phase 5 — Review Queue (Manual Columns)

These are the columns that make the Clay table the "brain" during the demo walk-through.

SettingValue

OptionsAPPROVE, REJECT, EDIT

This is the human review gate for Tier 1. Only Tier 1 rows should ever have APPROVE set. The Apps Script checks this column via the Send Queue Google Sheet.

Column type: Text. Leave empty to use the generated email body. If you edit the copy in Clay before approving, paste your version here — the Apps Script will use this over the generated body.

For the demo, set Acme Corp's send_status to queued in the Send Queue sheet BEFORE the Apps Script runs. Beta Tech and Gamma SaaS are already set to not_sent_mvp_watchlist.

Clay → Send Queue: How the Integration Works

On Clay's free tier, there is no REST API. The integration works through Google Sheets as the shared data layer.

The Send Queue Google Sheet tab is already pre-populated by QuadSci_SheetSetup.gs with all 3 demo accounts and their generated email copy (using the worked examples from the PRD). You do not need Clay to write to the sheet automatically for the demo.

During the live demo (§11 walkthrough):

  1. Show the Clay table — walk through Acme Corp's score (83), tier (Tier 1), email subject, email body.
  2. Set action = APPROVE in the Clay Action column (for show).
  3. Switch to the Google Sheets Send Queue tab — Acme Corp's row is already there with send_status = queued. Say: "In production, Clay writes approved rows here automatically. For the demo I've pre-staged the approved row."
  4. Open Apps Script → click Run → runDemoSendNow. The execution log appears immediately.
  5. Open Gmail Sent folder — email is there, addressed to your own inbox per demo-mode override.
Why this is better for the demo than a fully automated integration: You control the exact moment the send happens. No waiting for a 5-minute trigger. The narrative is cleaner: you show the Clay intelligence, you approve, you run the send, you show the receipt.

BuiltWith Substitution (Free Tier)

The PRD references BuiltWith via Clay for tech stack detection. BuiltWith is a paid Clay enrichment. For the demo, the tech stack is already pre-populated in the Accounts Google Sheet (tech_stack column). No BuiltWith call needed.

For a production build: Install the Wappalyzer browser extension (free). Visit each account's website and it shows their tech stack in real-time. Manually enter into the Clay table for the initial batch. For at-scale automation, Wappalyzer's free API tier provides 50 lookups/month.
