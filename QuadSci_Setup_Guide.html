Step 6 — Import All 6 Workflows into n8n (15 minutes)

⚠️ Import order matters. Import WF5 first so you have its webhook URL ready to paste into WF4.
A
How to import a workflow JSON file
  1. In n8n, click the + button (or "New Workflow")
  2. Click the three-dot menu (⋮) in the top-right of the canvas
  3. Select Import from File
  4. Choose the .json file from this folder
  5. The workflow appears on canvas — n8n will prompt you to reconnect any credentials it couldn't auto-match
  6. Click Save (Ctrl+S)
B
Import WF5 first — then copy its webhook URL
  1. Import wf5_copy_generator.json
  2. Click the Webhook — Generate Email node
  3. Copy the Webhook URL shown in the node panel (it starts with your n8n URL)
  4. Open wf4_ai_scorer.json in Notepad
  5. Find REPLACE_WITH_WF5_WEBHOOK_URL and replace it with the URL you just copied
  6. Save the file
✅ Now import WF4, then WF1, WF2, WF3, WF6 in any order.
C
After importing WF4 — register its webhook URL in Clay
  1. Import wf4_ai_scorer.json
  2. Click the Webhook — Clay Enrichment Done node
  3. Copy its webhook URL
  4. Go to Clay → Settings → Integrations → On Completion Webhook → paste the URL → Save

This tells Clay: "When you finish enriching a company, ping this URL." WF4 then wakes up and scores it.

Step 7 — Set Up Your Clay Table (15 minutes)

7
Build the Clay enrichment table
  1. Log in to clay.com → click + New Table
  2. Name it: QuadSci Account Enrichment
  3. Add these columns (Clay suggests enrichment sources automatically when you name them): Company Domain, Company Name, Employee Count, Funding Stage, Estimated ARR, Tech Stack, CS Headcount, Buyer Name, Buyer Title, Buyer Email, Email Verified
  4. For each column, click the header → choose an enrichment source (Clearbit, LinkedIn, etc.)
  5. Go to Settings → Integrations → Inbound Webhook → copy the webhook URL
  6. Open wf3_cluster_detector.json in Notepad → replace REPLACE_WITH_YOUR_CLAY_WEBHOOK_URL with this URL → save
💡 The inbound webhook is what WF3 sends to. When WF3 detects a cluster, it POSTs the company domain to Clay, which then automatically enriches it and (when done) calls WF4's webhook.

Step 9 — Test End-to-End Before Going Live (20 minutes)

9
Inject a fake signal and watch it flow through every stage

Before activating the schedules, do a manual test run with fake data.

  1. Open your Signal Log tab → add 3 rows with the same company domain, different signal types, dates within the last 21 days
  2. Go to n8n → open WF3 Cluster Detector → click Execute Workflow
  3. Check your Scored Accounts tab — the fake company should appear with status "Pending Enrichment"
  4. Manually fill in the enrichment columns for that row (fake values like 500 employees, Series B, etc.)
  5. Then open WF4 AI Scorer → click Execute → check that scores appear in Scored Accounts
  6. If tier is Tier 1 or Tier 2, check your Review Queue tab — an email draft should appear within 30 seconds
  7. Type APPROVE in the "Your Action" column
  8. Go to WF6 Approval Monitor → click Execute → check that Gmail sends the email and "Send Date" fills in
✅ If all 7 steps work, your pipeline is fully functional. Proceed to Step 10.
